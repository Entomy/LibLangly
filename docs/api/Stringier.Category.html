Remarks

While this has analogues to UnicodeCategory, there are a number of issues with how Microsoft approached that. First and foremost, there's no convenient way to group categories, even though UAX#44-5.7.1 does exactly this. This grouping of categories is important from a usability standpoint, for example, removing all symbols from a given text, or identifying any punctuation. Furthermore, UnicodeCategory strays from the names that the UNICODE Consortium gave, which is confusing, such as OtherNotAssigned versus Unassigned.

This has been designed with respect to Set Theory and Set Algebra, so far more complicated situations than UnicodeCategory can provide are possible through this.

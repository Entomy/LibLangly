Remarks

This fundamentally operates differently from . While it is designed fairly similar in API, the way it operates, even at that level, differs greatly. There's three major components that make up the byte stream:

This Stream type provides orchestration logic around the other components, tying everything together. Through those, it provides a low level, byte-centric, API.

The actual datastream is provided by StreamBase, which implements the basic streaming API for the daatastream. Supporting additional stream types can be done by extended StreamBase, which greatly simplifies adding new datastreams.

Buffers are provided by implementers of IReadBuffer and IWriteBuffer, which buffer the basic streaming from StreamBase, enabling features like Peek(out TElement) and limited ISeek behavior even when the underlying stream is not capable.

Additional orchestration can be added by deriving from this type. However, most applications will not need to do this. Support for additional types should be done through the serialization mechanism.

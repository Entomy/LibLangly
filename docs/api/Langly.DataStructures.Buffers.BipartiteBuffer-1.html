Remarks

This is a type of large buffer exhibiting ring-like behavior. It doesn't operate in the same way as a ring, keeping memory contiguous, which is a major advantage. It operates through partitioning the memory into two regions, advancing them like a ring-buffer would, but preventing breaking either partition. As a result, this buffer can return direct ReadOnlyMemory<T>, rather than allocate-fill-return.

Remarks

This uses an entire memory page as the buffer. This is by far the best for performance, and the most common use case for a bipartite buffer. It may, however, be way more memory than your application requires.

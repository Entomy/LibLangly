Remarks

A set is a collection in which any element can only exist once (all other collections are technically multi-sets, if you need a multi-set). Furthermore, sets don't care about data being ordered, although it may be ordered as an internal implementation details.

Unlike ISet<T> this adheres to Set Theory and Set Algebra, so its behavior should not be surprising, and existing techniques and solutions can be easily translated.

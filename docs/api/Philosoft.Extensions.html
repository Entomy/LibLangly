Clone<TSelf>(ICloneable<TSelf>)

Creates a new object that is a copy of the current instance.

Declaration
public static TSelf Clone<TSelf>(this ICloneable<TSelf> collection)
Parameters
Type Name Description
ICloneable<TSelf> collection

This collection.
Returns
Type Description
TSelf

A new object that is a copy of this instance.
Type Parameters
Name Description
TSelf

The type of this collection; itself.
Remarks

The resulting clone must be of the same type as, or compatible with, the original instance.

An implementation of Clone() can perform either a deep copy or a shallow copy. In a deep copy, all objects are duplicated; in a shallow copy, only the top-level objects are duplicated and the lower levels contain references. Because callers of Clone() cannot depend on the method performing a predictable cloning operation, we recommend that ICloneable<TSelf> not be implemented in public APIs.

See MemberwiseClone() for more information on cloning, deep versus shallow copies, and examples.

Map<TElement>(IIndexable<TElement>, Func<TElement, TElement>)

Applies the func to each element of the collection.

Declaration
public static void Map<TElement>(this IIndexable<TElement> collection, Func<TElement, TElement> func)
Parameters
Type Name Description
IIndexable<TElement> collection

This collection.
Func<TElement, TElement> func

The Func<T,TResult> to apply to each element.
Type Parameters
Name Description
TElement

The type of elements in the collection.
Remarks

This is a mutating operation. It mutates collection rather than constructing a new collection for the result. This is done for two reasons. First is that we can't generalize Map<TElement>(IIndexable<TElement>, Func<TElement, TElement>) like this while also allowing arbitrary collections to be constructed. Second is performance, as mutating in-place is substantially more efficient. Philosoft is not a pure functional programming library, it just takes inspiration in the form of more practical application of many functional principals.

Parameters

The identity value for func.
Returns
Type Description

A single element after folding the entire collection. If collection or func are null this function returns identity.
Type Parameters
Name Description
TElement

The type of elements in the collection.
TEnumerator

The enumerator for the collection.
Remarks

func is a magma, so associativity like left-fold and right-fold are completely irrelevant.

identity is required as a start point for the fold. It needs to be the identity of the func to function properly. For example, the identity of addition is 0, and the identity of multiplication is 1. Without an appropriate identity, the results will be wrong.

Map<TElement>(IIndexRef<TElement>, Func<TElement, TElement>)

Applies the func to each element of the collection.

Declaration
public static void Map<TElement>(this IIndexRef<TElement> collection, Func<TElement, TElement> func)
Parameters
Type Name Description
IIndexRef<TElement> collection

This collection.
Func<TElement, TElement> func

The Func<T,TResult> to apply to each element.
Type Parameters
Name Description
TElement

The type of elements in the collection.
Remarks

This is a mutating operation. It mutates collection rather than constructing a new collection for the result. This is done for two reasons. First is that we can't generalize Map<TElement>(IIndexRef<TElement>, Func<TElement, TElement>) like this while also allowing arbitrary collections to be constructed. Second is performance, as mutating in-place is substantially more efficient. Langly is not a pure functional programming library, it just takes inspiration in the form of more practical application of many functional principals.

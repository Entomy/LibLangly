public class TextStream : Stream, IDisposable, IAsyncDisposable, IPeekable<byte, Errors>, IReadable<byte, Errors>, ISeekable<byte, Errors>, IWritable<byte, Errors>, IPeekable<Rune, Errors>, IReadable<Rune, Errors>, ISeekable<Rune, Errors>, IWritable<Rune, Errors>
Remarks

This streams Rune instead of Char to avoid the misunderstandings and complexities working with UTF-16 Code Units causes. Rune represents a higher level that is closer to what people intend to work with.

This fundamentally operates differently from Stream. While it is designed fairly similar in API, the way it operates, even at that level, differs greatly. There's four major components that make up the text stream:

This TextStream type provides orchestration logic around the other components, tying everything together. Through those, it provides the higher level, text-centric, API.

The actual datastream is provided by StreamBase, which implements the basic streaming API for the daatastream.

Buffers are provided by IReadBuffer and IWriteBuffer, which buffer the basic streaming from StreamBase, enabling features like and limited Seekable behavior even when the underlying stream is not capable.

Encoding is handled by Encoder helpers. These handle the translation between encodings through a consistent API.

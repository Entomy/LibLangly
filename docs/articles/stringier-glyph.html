Glyph

The majority of programming languages, libraries, and tooling out there does not appropriately handle grapheme clusters. C# is one of these. What this means, in very simple terms, is that non-English languages will often be broken in ways that primarily English speakers will not recognize. This goes as far as many programs outright crashing for totally normal names. Glyph helps address this problem. The base class library provides StringInfo.GetTextElementEnumerator() and TextElementEnumerator, which perform a similar goal but with several issues. With the rise of Memory<T> and Span<T> we saw a lot of useful additions, both to performance and usability. However, Microsoft failed to adequately adapt many APIs to this purpose. Glyph helps handle this, by enumerating any textual type. Furthermore, it does so using a set of APIs that feel very similar to what .NET already provides with Char and Rune. The end result is an API that is more usable, performs better, and feels more familiar.

Using

using System.Text;

imports System.Text

open System.Text

Once you've imported the namespace, all functionality is exposes through Glyph and various extension methods. These are all analogous to working with Rune, so these examples should all be straightforward.

Enumerate grapheme clusters:

foreach (Glyph glyph in "tiếng Việt".EnumerateGlyphs()) {
	// Do something with each grapheme
}

For Each glyph in "tiếng Việt".EnumerateGlyphs
